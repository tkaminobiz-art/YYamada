## 1. 現状の課題整理

### 1-1. 画像の問題（最優先で解消）

| 現状の使用画像 | 問題 | 対応 |
|----------------|------|------|
| **山田氏本人/1.png, 2.png** | 本人確認済み | ✅ ヒーロー・プロフィールのみで使用継続 |
| **asm_question.jpg** | 男性演台スピーチ＝**山田洋平本人か未確認**。別人なら政治サイトルール違反 | ❌ 使用中止。本人写真に差し替え or 画像なし |
| **sty_family.jpg** | 家族4人・五重塔＝**本人家族か未確認**。ストック感 | ❌ 使用中止。本人・許諾済みに差し替え or 画像なし |
| **msg_children.jpg** | 公園の子ども＝肖像権・本人無関係 | △ 次世代「イメージ」としては可だが、**許諾確認**。不安なら外す |

| **act_inspection.jpg** | **女性・ヘルメット・建設現場**＝主役が別人 | ❌ 使用中止（政治サイト：依頼主以外が主役はNG） |

**鉄則（リニューアル後）:**  
- **人物が写っている画像は「山田洋平本人」または許諾済みのものだけ**とする。  
- 本人が主役でない写真・ストック風・外国人・透かし入りは**一切使わない**。  
- 該当する適切な画像がないセクションは**画像を置かず**、テキスト・数字・背景色・余白で構成する。

### 1-2. AIっぽさの原因（Anti-AI Aesthetic に基づく）

- **紋切り型:** 「バッジ3つ横並び → 説明1行 → LINE CTA 1つ」の繰り返し
- **均一な角丸・余白:** 8px / 16px など数値が揃いすぎ
- **安全色のみ:** 緑・オレンジ・グレーの組み合わせに「このサイトだけ」の差がない
- **ストック風・他人の写真:** 実在感・本人感が弱い
- **レイアウトの単調さ:** セクションごとに同じ型（見出し→カード列→CTA）

### 1-3. 構成・重複

- 参考スクショでは「ご挨拶」が二重に表示される不具合が指摘されている。  
- リニューアル後は**セクションの重複を一切つくらない**。

## 2. リニューアルの原則（5つ）

1. **画像は「本人 or 許諾済み」のみ。それ以外は画像なしで成立させる。**
2. **山下知事サイトを「構成・トーン」の正本とするが、見た目は紋切り型を避け、角丸・余白・色にゆらぎをつける。**
3. **CTAは「LINEボタン1本」にしない。QR・電話・短文・後援会など役割を分ける。**
4. **バッジ・タグは「3つ横並び」の型をやめる。1つにするか、形・色を意図的に変える。**
5. **HTML/CSSはシンプルに。余計なdiv・クラスを減らし、後から差し替えしやすい構造にする。**

## 3. 実装フェーズ

### Phase 0: 画像の整理（デザイン言語化のあと、実装前に実施）

- [ ] **使用許可リストを作る**  
  - 確定: `山田氏本人/1.png`, `山田氏本人/2.png` のみ。  
  - その他（sty_family, msg_children, asm_question, act_* 等）は**クライアントに「本人が写っているか・許諾済みか」を確認**。  
  - 返答がない or NG なら**一切使わない**。
- [ ] **リニューアル後の画像配置を決める**  
  - ヒーロー: 山田氏本人 1枚（1.png or 2.png）  
  - プロフィール: 山田氏本人 2枚（1.png, 2.png）  
  - ごあいさつ・政策・議会・活動: **本人写真 or 許諾画像**がなければ**画像ブロックなし**（テキスト・数字・背景のみ）。

### Phase 1: 構成・コンテンツの確定

- [ ] **セクション順を山下知事サイトに合わせる（1本化）**  
  1. ヘッダー（ロゴ＋ナビ）  
  2. ヒーロー（本人写真＋名前＋サブコピー）  
  3. 新着情報（ライトグリーンバー＋テキスト or リンク1〜3本。重複なし）  
  4. 動画（1本）＋ SNSリンク  
  5. ごあいさつ（**1回のみ**。本人 or 許諾画像があれば1枚、なければテキストのみ）  
  6. 実績（2大ボックス: 200万円 / 125万円＋説明）  
  7. 政策（01〜05 or 07。詳しく見るリンク。**画像は本人/許諾のみ or なし**）  
  8. 議会活動（テキスト＋Yohei's Check 等。**画像は本人/許諾のみ or なし**）  
  9. 活動（矢田山・辻町・視察など。**風景のみ or 本人が写っている写真のみ**）  
  10. プロフィール（本人写真2枚＋経歴・家族）  
  11. 後援会・LINE（CTA。ボタン＋QR or 短文。画像は使わない）  
  12. フッター（ダークグリーン・著作権・リンク）

- [ ] **重複セクションの削除**  
  - 「ごあいさつ」は1回だけ。  
  - 同一コピー・同一画像の繰り返しをコード上から排除する。

### Phase 2: 脱・AIっぽさ（デザインルールの明文化）

- [ ] **バッジ**  
  - 「3つ横並び」をやめる。  
  - 1つだけ強調するか、2つにして形を変える（例: 1つは枠線のみ、1つは塗り）。
- [ ] **CTA**  
  - 「LINEで活動報告を受け取る」単体にしない。  
  - 「後援会に入会する」・「お問い合わせ」・QRコード・電話のいずれかとセットで配置し、ブロックごとに役割を変える。
- [ ] **角丸・余白**  
  - 4px / 6px / 10px / 12px を用途で使い分ける。  
  - padding も 16px / 24px / 32px など揃えすぎない。
- [ ] **色**  
  - 緑一辺倒にしない。  
  - 1色だけ濃くする（例: ヘッダー・フッターをダークグリーン）か、アクセントに別トーンを1つ決める。
- [ ] **セクションの型**  
  - すべて「見出し中央→カード3列」にしない。  
  - 1列テキスト・2列（数字＋説明）・フル幅1枚・数字だけ大きく、などを意図的に混ぜる。
- [ ] **アイコン**  
  - SNSはアイコンで並べるなら1スタイルに統一。  
  - またはテキストリンク（X / Instagram / YouTube）のみにしてアイコンを減らす。

### Phase 3: コードのリニューアル

- [ ] **HTML**  
  - セマンティックに。`<main>`, `<section>`, `<article>`, `<header>`, `<footer>` を明確に。  
  - 余計な `<div>` を減らす。  
  - 画像は `<img>` のみ。背景画像はヒーロー1か所のみ（本人写真）。
- [ ] **CSS**  
  - 変数で色・余白・角丸をトークン化。  
  - クラス名は BEM 風（block__element--modifier）で整理。  
  - メディアクエリは 768px / 1024px の2段階で十分とするか、必要なら1段追加。
- [ ] **画像の参照**  
  - 使用する画像は `images/山田氏本人/` のみ、または許諾リストに載ったファイルのみ。  
  - それ以外のパスは書かない。
- [ ] **JS**  
  - ナビの開閉のみ。  
  - それ以外の動作は追加しない（リニューアル範囲を限定）。

### Phase 4: コンテンツ・コピーの最終確認

- [ ] 既存の「山田洋平_公式サイト_ページコピーライティング.md」をベースに、重複・冗長を削除。  
- [ ] 実績数字（200万円・125万円）はそのまま。  
- [ ] 政策は 01〜05 のままか、山下サイトに合わせ 01〜07 に拡張するかクライアント確認。  
- [ ] すべてのセクションで「ご挨拶」が二重になっていないか、公開前に1ページ全体を目視確認。

### Phase 5: 公開前チェックリスト

- [ ] 人物が写っている画像はすべて「山田洋平本人」または許諾済みか。  
- [ ] 外国人・海外画像・透かし入りは0枚か。  
- [ ] 「バッジ3つ→説明→CTA1つ」の型をそのまま繰り返していないか。  
- [ ] 角丸・余白にゆらぎがあるか。  
- [ ] 「ごあいさつ」セクションは1回だけか。  
- [ ] CTA が LINE 単体になっていないか。  
- [ ] ナビ・フッター・動画・SNS が正しく動作するか。
